# Smart Commits

Los SmartCommits nos ayudan a conservar un mejor orden al momento de documentar nuestros cambios, mejoras o correciones dentro del proyecto


#### Structure

`<ABBREVIATION> #<MODULE>: <COMMENTS..>`

Example: `[ADD] #settings: Adición de nuevos campos de configuración`

#### Abbreviation Commit
- `[FTR]` Implementación de una mejora
- `[ADD]` Adición de algun paquete, libreria o modulo nuevo
- `[UPD]` Actualizacion de código
- `[FIX]` Correción de código
- `[CLS]` Limpieza u optimización de código

#### Module Commit
- `#app` Toda la aplicacion
- `#workflow` Cambio en el flujo de trabajo
- `#overall` Cambios generales como .env.example, .npm, .git
- `#<module>` Se indica el nombre del modulo donde se realizó el cambio
